Elementos básicos

Los elementos principales que todo código HTML debería incluir son:

  1. Doctype Especifica la versión HTML utilizada. Debería emplearse HTML5 mediante la siguiente línea
    • <!DOCTYPE html>
  2. Idioma Se define dentro de la etiqueta html ("es"= español, "en"= inglés)
    • <html lang="es">
  3. Bloque head Debe contener los siguientes elementos
    1. Codificación Debe declararse lo antes posible
      • <meta charset="utf-8">
    2. Viewport es un elemento necesario que permite modificar el contenido de la web en función del tamaño de la pantalla donde se muestra
      • <meta name="viewport" content="width=device-width, initial-scale=1.0">
      • name="viewport" activa el método para tomar el control de la visualización de la web en la pantalla de los dispositivos.
      • width=device-width iguala el ancho de la web al de la pantalla del dispositivo donde se este mostrando.
      • initial-scale=1.0 establece el nivel inicial de zoom cuancuando la página es cargada por el navegador.
    3. Título Etiqueta requerida en HTML. Intentar ser lo más preciso y representativo posible ya que esta etiqueta:
      • Define el título a mostrar en la pestaña del navegador
      • Provee el nombre con el que se añadirá por defecto a favoritos
      • Utilizado en los mecanísmos de búsqueda
      • Se muestra en los resultados de búsqueda junto al resto de páginas
  4. Bloque body Contiene los elementos visibles en la web (e.g. títulos y encabezados, secciones, párrafos, imágenes, enlaces etc.)
    • La etiqueta más utilizada dentro de este bloque para organizar el contenido de la web son los divisores o contenedores <div>
Accesibilidad

Las etiquetas que agrupan los contenidos del body sirven también para mejorar su organización, facilitando la lectura de los screen readers y aplicaciones similares (e.g., sistemas de optimización de búsquedas - SEO). En definitiva, hacer una web accesible es facilitar la lectura del código, dotándolo de sentido semántico (semantic meaning), i.e., utilizar las etiquetas que hagan referencia al tipo de contenido o información que contienen.

In web development, accessibility refers to web content and a UI that can be understood, navigated, and interacted with by a broad audience. This includes people with visual, auditory, mobility, or cognitive disabilities.
Advertencia

En la documentación de HTML living standard se aconseja evitar una serie de etiquetas opcionales, entre las que se encuentran html y head. Sin embargo, en las guías de código del W3Consortium se recomienda no omitirlos debido a la existencia de posibles errores en navegadores antiguos.
