Formularios

Se utilizan en la interacción con los usuarios, obteniendo datos que pueden ser enviados a través del servidor. Pueden integrar distintos elementos, todos ellos dentro de la etiqueta <form></form>.

Elementos

A continuación se describen todos los elementos que pueden integrarse dentro de la etiqueta form, y que constituyen la interfaz gráfica de un formulario.

  • label
  • input
    • text
    • radio
    • checkbox
    • email
    • number
    • tel
    • time
    • button
  • textarea
  • select

<label> - Etiqueta o título de los elementos del formulario.

  • Es de utilidad para los usuarios con lector de pantallas, los cuales leerán en voz alta la etiqueta cuando se sitúen sobre el elemento al que hace referencia.
  • El atributo for de la etiqueta <label> debe ser igual al id del elemento al que esta asociada.

<input> - Etiqueta de auto cierre para obtener información del usuario. Existen diferentes tipos de input, llamados a través del atributo type:

  • text
    • Muestra un recuadro donde escribir cualquier tipo de texto.
    • Puede contener el atributo placeholder, que permite mostrar un texto dento del recuadro hasta que se escriba en él.
  • radio
    • Botón seleccionable asociado al texto del input.
    • Los botones se agrupan cuando contienen el mismo atributo name. De esta forma, al seleccionar un botón se deseleccionarán automáticamente los del mismo grupo.
    • La etiqueta <fieldset> integra visualmente el conjunto de repuestas. Debe utilizarse salvo cuando las opciones son auto-explicativas, e.g. la selección de género.
    • Puede añadirse un título al grupo mediante la etiqueta <legend>
    • Es recomendable englobar los input que contienen los botones entre etiquetas label
  • checkbox
    • Cuadrados seleccionables para preguntas con más de una respuesta.
    • Al igual que los botones, pueden integrarse en su propia etiqueta y agruparse mediante el mismo atributo name.
  • email
    • Se utiliza para introducir direcciones de correo. Valida automáticamente el texto evitando las direcciones que no coincidan con un correo electrónico.
  • number
    • Define un campo numérico donde solo pueden integrarse números.
    • Mediante los atributos min y max pueden acotarse las cifras a incluir.
  • date
    • Despliega un selector de fechas.
    • Importante: en Safari y en los buscadores más antiguos esta opción puede no estar disponible. Se cambia por defecto a la opción text. Por ello es recomendable incluir una referencia al estilo de fecha deseado en la etiqueta o en el placeholder.
  • tel
    • Campo específico para números de teléfono.
    • El atributo pattern puede utilizarse para delimitar el formato del número.
  • time
    • Permite seleccionar una hora (no un huso horario).
    • Dependiendo del navegador puede aparecer un selector de fechas.
  • button
    • Botón que permite enviar las respuestas del formulario al servidor, incluyendo el atributo type="submit", o resetear todas las respuestas, cambiando submit por reset.
    • El atributo action integra la URL de la web a la que se enviará la información de los input

<textarea> - Define un espacio donde se puede introducir texto.

  • El atributo rows especifica el número de filas visibles en el recuadro.
  • El atributo cols especifica el ancho del recuadro.
  • Sus dimensiones también pueden modificarse utilizando las propiedades CSS width y height.
  • El atributo form permite asociar un elemento textarea con un formulario del que no forma parte, escribiendo el id de este último.

<select> - Despliega una lista con las posibles opciones de respuesta.

  • Por defecto solo una opción puede ser seleccionada. Si se quiere habilitar la posibilidad de selección múltiple debe integrarse el atributo multiple dentro de la etiqueta select.
  • Los elementos a seleccionar se colocan dentro de <option>.
  • Por defecto aparece marcada la primera, pero puede seleccionarse una opción concreta integrando el atributo selected.
  • Una opción puede desabilitarse integrando el atributo disabled. Combinando esta opción con selected se crea una opción que no se puede seleccionar para mostrar un pequeño texto explicativo.
  • Otra posibilidad es utilizar el elemento <optgroup>, que hace las veces de enunciado o etiqueta.
  • Con el atributo size pueden especificarse el número de valores visibles en el desplegable.

Atributos

Los atributos más utilizados dentro del elemento form son los siguientes:

Atributo Descripción
action

Describe la acción del formulario cuando este es enviado.

Habitualmente los datos del formulario se envían a un servidor, e.g., <form action="/action_page.php">

En el código superior, los datos son enviados a un archivo llamado action_page.php. Este contiene un script que gestiona la información del formulario.
target

Especifica donde debe aparecer la respuesta que recibe el usuario una vez enviado. Puede tomar los siguientes valores:

  • _blank - Aparece en una nueva ventana
  • _self - Se muestra en la misma ventana
  • framename - La respuesta se muestra en el iframe seleccionado
method

Especifica el método HTTP utilizado al subir la info del formulario.

Puede enviarse como variables URL (con method="get", método por defecto) o como HTTP post transaction (with method="post").

Notas del método get

  • Añade los datos del formulario a la URL, en pares name/value
  • NUNCA debe usarse para enviar datos sensibles, pues podrán ser visibles en la URL
  • El límite de la URL son 2048 caracteres

Notas del método post

  • Añade los datos dentro del cuerpo de la petición HTTP (no es mostrado en la URL)
  • No tiene limitación de tamaño
  • Utilizar siempre que el formulario contenga datos de carácter personal
autocomplete Puede ser on u off. Si se activa (on), el navegador completará automáticamente los posibles valores basándose en las respuestas anteriores del usuario.
novalidate Es un atributo booleano, i.e., si aparece (<form novalidate>) se activa automáticamente. Cuando está presente, especifica que los inputs no deben ser validados cuando se envían.

Los input también contienen atributos específicos.

Los más utilizados son:

Atributo Descripción
required El formulario no podrá ser enviado hasta que dicho campo esté rellenado
disabled Input deshabilitado
checked El input aparece seleccionado por defecto (válido en type="checkbox" y type="radio")
maxlength y minlength Delimitan el número máximo y mínimo de caracteres que pueden escribirse
pattern Permite integrar una expresión regular con el formato permitido en la respuesta. Debe ir acompañado por el atributo title con una descripción del patrón.

Conexión formulario-servidor

El archivo de texto que se enviará al servidor contiene los enunciados de cada input, i.e., el texto dentro del atributo name.

Al lado de cada enunciado se escribe la respuesta, y en el caso de los inputs checkbox y radio, se envía el texto que contiene el atributo value de la opción seleccionada.

Si se selecciona la opción <input type=radio name="menu" value="pasta">Pasta se enviaría la siguiente línea: menu=pasta.
