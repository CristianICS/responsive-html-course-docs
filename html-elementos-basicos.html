Declarar lenguaje

La primera línea de un documento HTML debe ser la declaración <!DOCTYPE html>, que especifica HTML5 como lenguaje utilizado.

Idioma

Se define dentro de la etiqueta html ("es"= español, "en"= inglés)

  • <html lang="es">

Bloque <head>

Debe contener los siguientes elementos:

  1. Codificación Establece el tipo de codificación del documento. Debe declararse lo antes posible con la siguiente línea (cambiando el atributo charset dependiendo del tipo de codificación)
    • <meta charset="utf-8">
  2. Viewport Permite al navegador modificar las dimensiones de los elementos web en función del tamaño de la pestaña en la que se muestran, utilizando la siguiente línea
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">
    Donde:
    • name="viewport" activa el método para tomar el control de la visualización de la web
    • width=device-width iguala el ancho de la web al de la pantalla del dispositivo donde se este mostrando.
    • initial-scale=1.0 establece el nivel inicial de zoom cuancuando la página es cargada por el navegador.
  3. Título Etiqueta requerida en HTML. El título debe ser lo más preciso y representativo posible ya que esta etiqueta:
    • Define el título a mostrar en la pestaña del navegador
    • Provee el nombre con el que se añadirá por defecto a favoritos
    • Es utilizado en los mecanísmos de búsqueda de los buscadores
    • Se muestra en los resultados de búsqueda junto al resto de páginas
Advertencia

En la documentación de HTML living standard se aconseja evitar una serie de etiquetas opcionales, entre las que se encuentran html y head. Sin embargo, en las guías de código del W3Consortium se recomienda no omitirlos, debido a que pueden aparecer errores en navegadores antiguos.

Bloque <body>

Contiene los elementos de la web que se mostrarán en el navegador (e.g. títulos y encabezados, secciones, párrafos, imágenes, enlaces etc.)

Accesibilidad

Las etiquetas que agrupan los contenidos del body sirven también para mejorar su organización, facilitando la lectura de los screen readers y aplicaciones similares (e.g., sistemas de optimización de búsquedas - SEO). En definitiva, hacer una web accesible es facilitar la lectura del código, dotándolo de sentido semántico (semantic meaning), i.e., utilizar las etiquetas que hagan referencia al tipo de contenido o información que contienen.

In web development, accessibility refers to web content and a UI that can be understood, navigated, and interacted with by a broad audience. This includes people with visual, auditory, mobility, or cognitive disabilities.
